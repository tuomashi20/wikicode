## 📚 Wiki 知识文档草稿：`/memdraft` 模块

---

### 📌 模块名称  
`/memdraft` —— 临时记忆草稿管理模块

---

### 🧩 模块概述  
`/memdraft` 是 WikiCoder 架构中的一个轻量级、无状态的临时内容管理模块，用于创建、编辑、查看和保存临时草稿内容。该模块适用于需要快速记录、迭代和整理信息的场景，如撰写文档、整理会议纪要、规划项目等。

---

### 🧱 核心功能

| 功能 | 描述 |
|------|------|
| 创建草稿 | 用户可通过命令或接口创建新草稿，系统返回唯一 ID |
| 查看草稿 | 支持查看所有草稿列表或指定 ID 的草稿内容 |
| 编辑草稿 | 支持对已有草稿进行内容修改 |
| 保存草稿 | 将草稿内容保存为正式知识条目（如 Wiki 页面） |
| 删除草稿 | 支持删除不再需要的草稿 |
| 自动过期 | 草稿在设定时间后自动清除，避免内存堆积 |

---

### 📦 数据结构设计

```json
{
  "id": "string",           // 唯一标识符
  "title": "string",        // 草稿标题
  "content": "string",      // 草稿内容（支持 Markdown）
  "createdAt": "timestamp", // 创建时间
  "lastModified": "timestamp", // 最后修改时间
  "status": "draft|saved",  // 草稿状态
  "ttl": "number"           // 生存时间（毫秒）
}
```

---

### 🧩 接口规范（RESTful）

| 方法 | 路径 | 描述 |
|------|------|------|
| `POST` | `/memdraft` | 创建新草稿，返回 ID |
| `GET` | `/memdraft` | 获取所有草稿列表 |
| `GET` | `/memdraft/:id` | 获取指定草稿内容 |
| `PUT` | `/memdraft/:id` | 更新草稿内容 |
| `DELETE` | `/memdraft/:id` | 删除草稿 |
| `POST` | `/memdraft/:id/save` | 将草稿保存为正式知识条目 |

---

### 🧠 存储机制

- **存储方式**：使用内存缓存（如 Redis）或本地临时数据库（如 SQLite）
- **自动清理**：设置 TTL（Time To Live），超时后自动清除草稿
- **会话绑定**：草稿 ID 可与用户会话（session）或临时 token 绑定，确保隔离性

### 🧩 模块概述

`/memdraft` 是 WikiCoder 系统中的一个轻量级、临时内容管理模块，用于支持用户在不立即持久化的情况下创建、编辑和管理草稿内容。该模块适用于快速记录、临时协作、内容预览等场景，是“思考-记录-整理-执行”工作流中的关键一环。

---

### 🎯 核心功能

| 功能 | 描述 |
|------|------|
| **创建草稿** | 用户可通过命令或接口创建临时草稿，系统返回唯一 ID。 |
| **查看草稿** | 支持列出所有草稿或通过 ID 查看具体内容。 |
| **编辑草稿** | 支持对已有草稿进行修改、追加或删除操作。 |
| **保存草稿** | 可将草稿内容提交为正式知识条目（如 Wiki 页面、笔记等）。 |
| **删除草稿** | 支持手动删除或自动清理过期草稿。 |
| **自动过期** | 草稿在设定时间后自动清除，避免内存堆积。 |
| **会话绑定** | 草稿与用户会话绑定，确保数据隔离与安全性。 |

### 🧱 数据结构设计

```ts
interface Draft {
  id: string;           // 唯一标识符
  title?: string;       // 草稿标题（可选）
  content: string;      // 草稿内容（支持 Markdown 或纯文本）
  createdAt: number;    // 创建时间戳（毫秒）
  lastModified: number; // 最后修改时间戳（毫秒）
  ttl: number;          // 生存时间（毫秒）
  status: 'draft' | 'saved'; // 草稿状态
}
```

### 🛠️ 接口规范（RESTful）

| `GET` | `/memdraft` | 列出所有草稿（可分页） |
| `GET` | `/memdraft/:id` | 获取指定草稿内容 |
| `PUT` | `/memdraft/:id` | 更新草稿内容 |
| `DELETE` | `/memdraft/:id` | 删除草稿（或标记为过期） |

---

| `/memdraft list` | - | 列出所有草稿 |
| `/memdraft view <id>` | ID | 查看指定草稿 |
| `/memdraft edit <id> <content>` | ID, 内容 | 编辑草稿内容 |
| `/memdraft save <id>` | ID | 将草稿保存为正式知识条目 |
| `/memdraft delete <id>` | ID | 删除草稿 |

### 🧠 实现要点

#### 1. **内存存储结构**
- 使用 `Map<string, Draft>` 存储草稿内容。
- 每个草稿对象包含内容、时间戳、TTL 等字段。

#### 2. **自动过期机制**
- 启动后台定时任务（如每分钟扫描一次），清理过期草稿。
- 或使用 Redis 的 `EXPIRE` 命令实现 TTL 自动清除。

#### 3. **会话管理**
- 草稿 ID 与用户会话（session）或临时 token 绑定。
- 支持通过 cookie 或 header 传递会话标识。

#### 4. **前端交互逻辑（可选）**
- 实现自动保存（auto-save）功能，定时将编辑器内容同步到 `/memdraft`。
- 提供“恢复草稿”按钮，从内存中加载未提交内容。
