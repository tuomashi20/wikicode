# 递归式项目记忆管理规范 (Recursive Memory Indexing)

当项目规模扩大、文件增多时，为了防止“上下文迷失”，你必须主动维护项目的长程记忆索引。

## 1. 记忆锚点文件：`.wikicoder_memory.md`
在项目根目录下，你应当主动创建并维护一个名为 `.wikicoder_memory.md` 的文件。它充当你的“项目白板”和“知识索引”。

## 2. 必须记录的核心内容
每当你完成一个重要的阶段性任务（如：创建了数据库模型、配置了鉴权中间件、重构了核心 API），你应当更新该文件，记录以下信息：
- **【模块索引】**：记录核心功能的入口文件及关联文件路径。
- **【架构决议】**：记录你选择某种特定实现方式的原因（如：为何选择 Redis 做缓存而非内存）。
- **【排坑日记】**：记录你在该项目中遇到的特定环境问题及对应的解决方案，防止后续步骤重复踩坑。

## 3. 检索与自愈习惯
- **先读索引，再动刀**：在接手一个已有项目或长周期任务时，第一步应当是 `cat .wikicoder_memory.md`。
- **动态更新**：禁止让记忆文件过时。如果你发现之前的记忆有误或逻辑已变更，必须立即执行 `edit_file` 更新它。

> 记住：`.wikicoder_memory.md` 是你的“第二大脑”。当执行历史被快照折叠后，这里是你找回项目细节的唯一途径。
