# 工业级工程核验与自愈规范 (General Engineering Rules)

为了确保交付代码的高质量与可运行性，你必须在执行任务时严格遵守以下“自愈闭环”流程：

## 1. 交付前强制校验原则 (Verify-Before-Finish)
在你宣告任务完成（执行 `finish`）之前，如果在此过程中你修改过任何源代码或配置文件，你**必须**主动执行相关的环境校验指令：
- **Next.js/React 项目**：执行 `npm run lint` 或 `npx next build`。
- **Python 项目**：执行 `python -m py_compile <文件名>` 或运行相关的单元测试脚本。
- **通用项目**：确保执行了相关的编译或静态检查命令。

## 2. 报错回流与深度自愈 (Error Feedback Loop)
如果校验指令返回了非零退出码或错误日志：
- **禁止视而不见**：绝对禁止在明知有编译错误的情况下调用 `finish`。
- **逻辑重构**：将报错信息视为最高优先级的 `Observation`。你必须在 `self_criticism` 中分析报错原因（如：语法错误、类型冲突、路径引用错误等），并立即发起下一次 `edit_file` 进行修复。
- **重复校验**：修复后，必须**再次执行**校验指令，直到结果为 `Success`。

## 3. 宁可多跑一步，不可错漏一行
如果你不确定某个修改是否影响了全局，请优先运行项目的“冒烟测试”或启动开发服务器（`npm run dev`）并通过 `read_url` 或 `shell` 检查其状态。

> 记住：一个能够通过编译器审计的代码，才是真正的交付成果。
